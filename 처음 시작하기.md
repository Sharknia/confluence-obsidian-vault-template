# 처음 시작하기

이 vault는 Confluence 문서를 Obsidian에서 편집 가능한 Markdown 작업 사본으로 다루기 위한 기본 템플릿입니다.

## 1. Obsidian에서 열기

Obsidian에서 `Open folder as vault`를 선택하고 이 폴더를 엽니다.

## 2. 플러그인 활성화

`Settings > Community plugins`로 이동합니다.

Restricted mode를 끈 뒤 `Confluence Obsidian Sync`를 활성화합니다.

## 3. 연결 정보 입력

플러그인 설정에서 아래 항목을 입력합니다.

- Confluence base URL
- Atlassian account email
- API token

API token은 Atlassian 계정 보안 설정에서 발급합니다.

## 4. 프로젝트 생성

`Root content URL`에는 기본값으로 `https://selta.atlassian.net/wiki/spaces/IS/folder/23167000`이 입력되어 있습니다.

다른 루트 페이지 또는 루트 폴더를 사용할 때만 URL을 바꾼 뒤 `Create project`를 실행합니다.

## 5. 문서 내려받기

왼쪽 리본 아이콘 또는 명령 팔레트에서 `Open Sync Panel`을 실행한 뒤 `Pull Tree`를 누릅니다.

문서는 `confluence/` 폴더에 저장됩니다.

## 6. 결과 확인

최근 Pull 상세 결과는 `logs/latest.md`에서 확인합니다.

graphify CLI를 별도로 설치한 경우 Sync Panel에서 현재 프로젝트를 분석하고 `graphify-out/` 폴더의 결과를 확인할 수 있습니다.

## 7. 터미널과 업데이트

Sync Panel의 `터미널 열기`는 이 vault 루트를 작업 폴더로 터미널을 엽니다.

Sync Panel의 `플러그인 업데이트`는 GitHub 최신 Release에서 플러그인 파일만 교체합니다. 설정 파일은 유지됩니다.
